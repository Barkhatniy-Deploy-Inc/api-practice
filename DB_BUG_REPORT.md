# Отчет об аудите базы данных (DTP_DB.db)

В ходе интеграции Backend API с текущей версией базы данных выявлены критические ошибки в структуре и отсутствие данных.

## 🔴 Критические ошибки (Блокируют работу API)

1. **Таблица `vehicles`**: Отсутствует связь с ДТП.
   - **Проблема**: Нет колонки `accident_id`. Мы не можем сопоставить машину и аварию.
   - **Решение**: `ALTER TABLE vehicles ADD COLUMN accident_id INTEGER REFERENCES accidents(id) ON DELETE CASCADE;`

2. **Таблица `districts`**: Отсутствуют коды округов.
   - **Проблема**: В ТЗ и API требуются коды ("ЦФО", "ЮФО" и т.д.) для фильтрации и аналитики. Сейчас есть только `id` и `name`.
   - **Решение**: Добавить колонку `code TEXT UNIQUE`.

3. **Таблица `participants`**: Жесткое ограничение `NOT NULL` на поле `experience`.
   - **Проблема**: У пешеходов или пассажиров нет водительского стажа. API падает при попытке записать `NULL`.
   - **Решение**: Снять ограничение `NOT NULL` или сделать стаж `DEFAULT 0`.

## 🟡 Несоответствия спецификации

1. **Поле `gender`**: В БД это `TEXT`, в ТЗ ожидался `Boolean` (1-М, 0-Ж). Если оставляем текст, нужно зафиксировать допустимые значения ("Мужской"/"Женский").
2. **Таблица `brief_info`**: Кажется избыточной, так как все данные уже связаны через `accidents`. Просьба уточнить назначение.

## ⚪ Статус данных
- **Текущее состояние**: 0 записей во всех таблицах.
- **Просьба**: Наполнить справочники (`regions`, `districts`, `weather_types`, `accident_types`) и добавить хотя бы 10-20 тестовых строк в `accidents`, чтобы мы могли проверить формулы аналитики.
