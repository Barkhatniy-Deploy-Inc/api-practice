# Зебра-стат: Регламент развертывания и эксплуатации (DOD)

Этот документ предназначен для DevOps-инженера и содержит все технические детали для запуска Зебра-стат API в продакшн-среде.

---

## 1. Стек технологий
*   **Язык:** Python 3.10+
*   **Фреймворк:** FastAPI (ASGI)
*   **База данных:** SQLite (локальный файл `dtp.db`)
*   **Контейнеризация:** Docker + Docker Compose

---

## 2. Переменные окружения
Система требует настройки следующих параметров:

| Переменная | Описание | Обязательно |
|------------|----------|-------------|
| `X_API_KEY` | Мастер-ключ (мин. 32 символа) для управления API. | Да |
| `AUTH_SALT` | Соль для хеширования студенческих ключей. При смене ключи станут недействительны. | Да |
| `DATABASE_URL` | URL базы данных (в Docker: `sqlite+aiosqlite:////app/data/dtp.db`). | Нет |
| `LOG_LEVEL` | Уровень логирования (DEBUG, INFO, WARNING, ERROR). | Нет |
| `DEBUG_MODE` | Режим отладки (true/false). | Нет |

---

## 3. Быстрый запуск (Docker)
Для развертывания одной командой выполните:
```bash
# Укажите ваши ключи в .env файле (см. .env.example)
docker-compose up -d --build
```

---

## 4. Сбор логов и мониторинг
*   **Логи контейнера:** выводятся в `stdout` (рекомендуется настроить драйвер `json-file` или `gelf`).
*   **Файловые логи:** монтируются в локальную папку `./logs` из контейнера.
*   **Health Check:** доступен по эндпоинту `GET /health`. Возвращает `{"status":"ok"}`.

---

## 5. Обслуживание базы данных
Поскольку используется SQLite, для резервного копирования достаточно делать snapshot файла `dtp.db`. При миграции на PostgreSQL потребуется обновление `app/core/config.py`.

---

## 6. Автоматизация (CI/CD)
В репозитории настроен GitHub Actions (`.github/workflows/ci.yml`). При каждом пуше в ветки `main/master` запускается полный цикл тестов (56 тестов) с замером покрытия.

**Критерий качества:** покрытие тестами должно быть выше 80% (текущее: 88%).
