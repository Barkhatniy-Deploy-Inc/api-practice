# Спецификация логирования действий (Action Logging)

Данный документ описывает систему аудита бизнес-логики и\u00A0пользовательских действий в\u00A0рамках Зебра-стат API.

---

## 1. Суть системы
В\u00A0отличие от\u00A0технического логирования подключений, система логирования действий фиксирует **смысловую нагрузку** запроса. Мы\u00A0отслеживаем, какие именно исследования проводит студент, чтобы оценить его работу с\u00A0наборами данных ГИБДД.

---

## 2. Логируемые события
Система регистрирует следующие типы действий:
*   **Чтение данных:** получение списка ДТП с\u00A0указанием примененных фильтров (например, «ДТП с\u00A0детьми» или\u00A0«пьяные водители»).
*   **Генерация аналитики:** запросы к\u00A0математическим моделям (тренды, прогнозы, расчеты рисков).
*   **Работа со\u00A0справочниками:** изучение структуры классификаторов ГИБДД.
*   **Авторизация:** успешное прохождение проверки ключа и\u00A0идентификация владельца.

---

## 3. Детализация параметров (Payload Logging)
Для\u00A0каждого действия в\u00A0лог записываются входные параметры, имеющие значение для\u00A0исследования:
*   `query_params`: выбранные регионы, диапазоны дат, лимиты выборок.
*   `filters_applied`: флаги фильтрации (`has_children`, `is_drunk`).
*   `execution_time`: время обработки запроса сервером (в\u00A0секундах).

---

## 4. Пример структуры действия
Запись в\u00A0логе при\u00A0запросе рейтинга безопасности:

```json
{
  "timestamp": "2026-04-22 19:10:05,442",
  "level": "INFO",
  "action": "generate_report",
  "report_type": "safety_rating",
  "student": "Петр Петров",
  "context": {
    "sort_by": "severity_index",
    "order": "desc",
    "rows_returned": 85
  },
  "performance": {
    "db_latency": "0.045s",
    "total_latency": "0.052s"
  }
}
```

---

## 5. Аналитическая ценность
Данные логирования действий позволяют:
1.  Выявлять наиболее популярные среди студентов наборы данных.
2.  Обнаруживать неэффективные поисковые запросы (например, слишком широкие выборки без\u00A0фильтров).
3.  Подтверждать самостоятельность выполнения лабораторных работ.

---

## 6. Хранение
Логи действий сохраняются в\u00A0отдельный файл `logs/api_actions.log` с\u00A0ежедневной ротацией. Файлы доступны администратору системы для\u00A0последующей выгрузки в\u00A0Excel или\u00A0Obsidian.
