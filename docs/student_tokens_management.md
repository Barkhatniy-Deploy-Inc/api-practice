# Руководство по\u00A0управлению студенческими токенами (Student Tokens Management)

В\u00A0этом документе описан регламент выдачи, сопровождения и\u00A0отзыва API-ключей для\u00A0участников образовательного процесса.

---

## 1. Процесс генерации ключей
Выдача ключей производится администратором системы (преподавателем). Рекомендуется придерживаться следующего алгоритма:
1.  Сформируйте список имен студентов (или\u00A0названий проектных групп).
2.  Используйте метод `POST /api/v1/auth/keys` для\u00A0каждого участника.
3.  Обязательно передавайте поле `owner_name` для\u00A0последующей идентификации действий студента в\u00A0логах.

---

## 2. Передача ключа студенту
**Внимание!** Поле `access_key` в\u00A0ответе сервера отображается **всего один раз**.
*   Скопируйте значение ключа сразу после генерации.
*   Передайте ключ студенту по\u00A0защищенному каналу связи.
*   Инструктируйте студента не\u00A0передавать свой ключ третьим лицам и\u00A0не\u00A0публиковать его в\u00A0открытых репозиториях (например, в\u00A0публичном GitHub).

---

## 3. Реестр выданных токенов
Для\u00A0просмотра текущего состояния всех ключей используйте метод:
`GET /api/v1/auth/keys` (требует Master-Key).
В\u00A0ответе вы\u00A0увидите:
*   `owner_name`: имя владельца.
*   `prefix`: префикс ключа (для\u00A0сверки с\u00A0логом действий).
*   `is_active`: текущий статус доступа.
*   `created_at`: дату выдачи.

---

## 4. Регламент отзыва доступа
Доступ студента к\u00A0API может быть прекращен в\u00A0следующих случаях:
1.  Завершение учебного курса.
2.  Компрометация ключа (утечка в\u00A0сеть).
3.  Нарушение правил использования API (попытки дестабилизации сервера).

Для\u00A0отзыва ключа необходимо изменить поле `is_active` на\u00A0`false` в\u00A0базе данных `dtp.db` (таблица `api_keys`) по\u00A0соответствующему префиксу.

---

## 5. Ответственность сторон
*   **Администратор:** гарантирует сохранность Master-ключа и\u00A0корректность привязки имен к\u00A0токенам.
*   **Студент:** несет ответственность за\u00A0все действия, совершенные под\u00A0его API-ключом. Все действия логируются и\u00A0подшиваются к\u00A0личному делу студента.
