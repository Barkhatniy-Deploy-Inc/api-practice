# Спецификация системы логирования (Logging Specification)

В этом документе описана архитектура сбора логов и форматы сообщений Зебра-стат API.

---

## 1. Архитектура логов
Система использует двухуровневое логирование:
1.  **Консольный вывод (Stdout):** для мониторинга в\u00A0реальном времени при разработке.
2.  **Файловое хранилище:** ротируемые JSON-логи в\u00A0папке `logs/` для долгосрочного хранения.

### Параметры ротации:
*   **Файл:** `logs/api_access.log`;
*   **Макс. размер:** 10\u00A0МБ;
*   **Количество бэкапов:** 5\u00A0последних файлов.

---

## 2. Формат сообщений (Structured Logging)
Все сообщения записываются в\u00A0формате JSON для легкого парсинга (например, через `jq`).

```json
{
  "timestamp": "2026-04-22 15:33:51,299",
  "level": "INFO",
  "logger": "api.core",
  "message": "User Sk_dtp_... accessed /api/v1/accidents"
}
```

---

## 3. Уровни логирования
*   **DEBUG:** подробная информация для отладки (SQL-запросы).
*   **INFO:** основные события (запуск сервера, успешные запросы).
*   **WARNING:** подозрительные действия (неверные API-ключи).
*   **ERROR:** ошибки в\u00A0коде или базе данных.
