# Спецификация системы логирования (Logging Specification)

В данном документе описывается архитектура сбора логов и форматы сообщений DTP Analytics API.

---

## 1. Архитектура логов
Система использует двухуровневое логирование:
1.  **Консольный вывод (Stdout):** Для мониторинга в реальном времени при разработке.
2.  **Файловое хранилище:** Ротируемые JSON-логи в папке `logs/` для долгосрочного хранения.

### Параметры ротации:
*   **Файл:** `logs/api_access.log`
*   **Макс. размер:** 10 МБ.
*   **Количество бэкапов:** 5 последних файлов.

---

## 2. Формат сообщений (Structured Logging)
Все сообщения записываются в формате JSON для легкого парсинга (например, через `jq`).

```json
{
  "timestamp": "2026-04-22 15:33:51,299",
  "level": "INFO",
  "logger": "api.core",
  "message": "User Sk_dtp_... accessed /api/v1/accidents"
}
```

---

## 3. Уровни логирования
*   **DEBUG:** Подробная информация для отладки (SQL-запросы).
*   **INFO:** Основные события (запуск сервера, успешные запросы).
*   **WARNING:** Подозрительные действия (неверные API-ключи).
*   **ERROR:** Ошибки в коде или базе данных.
