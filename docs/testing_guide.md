# Инструкция по тестированию API (API Testing Guide)

В данном документе приведены правила и примеры тестирования функциональности DTP Analytics API.

---

## 1. Стратегия тестирования
Для обеспечения высокого качества образовательного API используется трехуровневая модель:
1.  **Unit-тесты:** Проверка отдельных функций (математика регрессии, расчеты рисков).
2.  **Интеграционные тесты:** Проверка взаимодействия эндпоинтов с базой данных SQLite.
3.  **Smoke-тесты:** Проверка доступности основных ручек после деплоя.

---

## 2. Инструментарий
*   **pytest** — основной фреймворк для запуска тестов.
*   **httpx** — асинхронный клиент для запросов к FastAPI.
*   **curl / httpie** — для ручной проверки из терминала.

---

## 3. Примеры ручных тестов

### Проверка здоровья системы
```bash
curl -i -X GET "http://127.0.0.1:8000/health"
```

### Доступ к реестру ДТП (с авторизацией)
```bash
curl -H "X-API-KEY: ваш_токен" \
     -X GET "http://127.0.0.1:8000/api/v1/accidents/?limit=5"
```

### Проверка аналитики
```bash
curl -H "X-API-KEY: ваш_токен" \
     -X GET "http://127.0.0.1:8000/api/v1/analytics/trends/monthly"
```

---

## 4. Ожидаемые результаты
*   **200 OK**: Запрос успешен, данные в формате JSON.
*   **403 Forbidden**: Ключ не передан или неверен.
*   **422 Unprocessable Entity**: Ошибка валидации (например, передана строка вместо даты).
*   **500 Internal Server Error**: Ошибка на стороне сервера (сообщить разработчику).

---

## 5. Автоматизированный запуск
Выполните команду в корне проекта:
```bash
pytest tests/
```
Все тесты должны завершиться со статусом `PASSED`.
