## 0. Открытый реестр ДТП

Чтобы наполнить базу, используется официальный открытый реестр Госавтоинспекции:

| Источник | URL / API |
|---|---|
| **Открытые данные ГИБДД** (карточки ДТП) | `https://stat.gibdd.ru` → раздел «Открытые данные» |
| **REST API ГИБДД** (opendataapi v1) | `https://stat.gibdd.ru/opendataapi/v1/kartdtp` |
| **Росстат** (население для Social Risk) | `https://rosstat.gov.ru/opendata` |
| **ФГБУ «Росавтодор»** (реестр трасс) | `https://rosavtodor.gov.ru/opendata` |

> API ГИБДД возвращает JSON с\u00A0полями `empt_number`, `dtpv`, `coord_w`, `coord_l`, `pog`, `ran`, `ts_info`\u00A0— структура полностью соответствует таблице `accidents` ниже.

### 1.1 Справочные (Dictionary) таблицы

#### `districts`\u00A0— федеральные округа
> Нужна для\u00A0ручек №\u00A01, №\u00A018.

```sql
CREATE TABLE districts (
    id   INTEGER PRIMARY KEY AUTOINCREMENT, -- номер округа.
    name TEXT    NOT NULL UNIQUE    -- "Центральный федеральный округ"
);
```

---

#### `regions`\u00A0— субъекты РФ
> Нужна для\u00A0ручек №\u00A01, №\u00A09, №\u00A025, №\u00A026, №\u00A028, №\u00A038.

```sql
CREATE TABLE regions (
    code        TEXT    PRIMARY KEY,            -- "77", "50", "02" …
    name        TEXT    NOT NULL,               -- "гор. Москва"
    district_id INTEGER NOT NULL
                REFERENCES districts(id),
    population  INTEGER,                        -- кол-во жителей (Росстат) — для Social Risk (#30)
    vehicles    INTEGER                         -- кол-во зарегистр. ТС — для Transport Risk (#31)
);
```

### 1.2 Основные (Fact) таблицы

#### `accidents`\u00A0— центральный реестр ДТП ⭐
> Ядро системы. Покрывает ручки №\u00A07–20, 21–24, 27–32, 34–40.

```sql
CREATE TABLE accidents (
    -- Идентификация
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    empt_number     TEXT    NOT NULL UNIQUE,    -- системный номер ГИБДД, напр. "140000008"

    -- Тип происшествия
    accident_type_id INTEGER REFERENCES accident_types(id),

    -- Временные метки (для ручек: #10, #15, #21, #22, #23, #24, #36)
    date_dtp        DATE    NOT NULL,           -- ГГГГ-ММ-ДД
    time_dtp        TIME    NOT NULL,           -- ЧЧ:ММ
    hour_dtp        INTEGER GENERATED ALWAYS AS (CAST(strftime('%H', time_dtp) AS INTEGER)) VIRTUAL,
    day_of_week     INTEGER GENERATED ALWAYS AS (CAST(strftime('%w', date_dtp) AS INTEGER)) VIRTUAL,
    is_weekend      INTEGER GENERATED ALWAYS AS (
                        CASE WHEN strftime('%w', date_dtp) IN ('0','6') THEN 1 ELSE 0 END
                    ) VIRTUAL,
    is_holiday      BOOLEAN DEFAULT 0, -- вручную или из календаря праздников

    -- География (для ручек: #9, #18, #19, #20, #27, #40)
    region_code     TEXT    NOT NULL REFERENCES regions(code),
    coord_lat       REAL,                       -- широта  (coord_w из API)
    coord_lon       REAL,                       -- долгота (coord_l из API)
    locality        TEXT,                       -- населённый пункт (np)
    road_name       TEXT,                       -- название дороги (dor)
    road_km         INTEGER,                    -- километр дороги
    is_city         BOOLEAN DEFAULT 1,          -- 1=в городе, 0=трасса (#19)
    road_category   TEXT,                       -- "Федеральная", "Региональная", "Местная" (#20)
    is_railway      BOOLEAN DEFAULT 0,          -- ДТП на ж/д переезде

    -- Условия (для ручек: #34, #35, #36, #37)
    weather_id      INTEGER REFERENCES weather_types(id),
    road_cond_id    INTEGER REFERENCES road_conditions(id),
    lighting        TEXT,                       -- "Светлое время", "Тёмное (освещение есть)", "Тёмное (нет)" (#36)
    has_road_defect INTEGER NOT NULL DEFAULT 0, -- дефект дороги (#37)
    road_type TEXT,                             -- Тип дороги "Шоссе", "Проспект", "Тракт", "Тупик"
    road_defect_desc TEXT,                      -- описание дефекта

    -- Последствия (для ручек: #16, #17, #29, #30, #31, #32)
    fatalities      INTEGER NOT NULL DEFAULT 0,    -- pog
    injured         INTEGER NOT NULL DEFAULT 0,    -- ran
    vehicles_count  INTEGER NOT NULL DEFAULT 1,    -- k_ts
    participants_count INTEGER NOT NULL DEFAULT 1, -- k_uch

    -- Специфические индикаторы (для ручек: #11, #13, #14)
    driver_fled     BOOLEAN DEFAULT 0,          -- водитель скрылся (#14) 1/0 (да/нет)
    has_children    BOOLEAN DEFAULT 0,          -- есть несовершеннолетние (#12)1/0 (да/нет)
    has_drunk       BOOLEAN DEFAULT 0,          -- есть нетрезвые (#13) — денормализация для быстрого фильтра. 1/0 (да/нет)

    -- Медицина (для ручки #39, отчёт #38)
    rescue_time_min INTEGER,                    -- время прибытия спецслужб в минутах

    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Индексы для ускорения частых запросов
CREATE INDEX idx_acc_date       ON accidents(date_dtp);
CREATE INDEX idx_acc_region     ON accidents(region_code);
CREATE INDEX idx_acc_city       ON accidents(is_city);
CREATE INDEX idx_acc_road_cat   ON accidents(road_category);
CREATE INDEX idx_acc_weather    ON accidents(weather_id);
CREATE INDEX idx_acc_fled       ON accidents(driver_fled);
CREATE INDEX idx_acc_children   ON accidents(has_children);
CREATE INDEX idx_acc_drunk      ON accidents(has_drunk);
CREATE INDEX idx_acc_coords     ON accidents(coord_lat, coord_lon);  -- для hotspots (#40)
```

#### `vehicles`\u00A0— транспортные средства в\u00A0ДТП

```sql
CREATE TABLE vehicles (

    car_model_id INTEGER REFERENCES car_models(id),
    year_release INTEGER,                       -- год выпуска ТС

);

CREATE INDEX idx_veh_accident ON vehicles(accident_id);
```

#### `participants`\u00A0— участники ДТП
> Нужна для\u00A0ручек №\u00A012, 13, 33, 35, 39\u00A0(портрет виновника).

```sql
CREATE TABLE participants (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    accident_id   INTEGER NOT NULL REFERENCES accidents(id) ON DELETE CASCADE,
    vehicle_id    INTEGER REFERENCES vehicles(id),    -- NULL для пешеходов

    -- Роль
    role          TEXT    NOT NULL,                   -- "Водитель", "Пешеход", "Пассажир", "Велосипедист"
    is_culprit    INTEGER NOT NULL DEFAULT 0,         -- виновник ДТП

    -- Демография (для ручек: #33, #39)
    gender        TEXT,                            -- муж, жен
    age           INTEGER,                            -- возраст (#33)
    experience    INTEGER,                            -- стаж вождения в годах (#35)

    -- Состояние
    is_drunk      BOOLEAN,                               -- алкогольное опьянение (#13)
    is_intoxication BOOLEAN,                             -- наркотическое опьянение (#13)
    alcohol_level REAL,                               -- промилле (если замерялось)
    before_health TEXT,                               -- на момент аварии/до аварии "имел сердечный приступ что вызвало аварию"
    health_status TEXT,                               -- "Погиб", "Тяжкий вред", "Лёгкий вред", "Без вреда"
    first_aid     INTEGER NOT NULL DEFAULT 0          -- оказана первая помощь (#39)
	"status comment"	TEXT                          -- комментарий состяния здоровья. 
);

CREATE INDEX idx_part_accident  ON participants(accident_id);
CREATE INDEX idx_part_role      ON participants(role);
CREATE INDEX idx_part_drunk     ON participants(is_drunk);
CREATE INDEX idx_part_age       ON participants(age);
```

#### `Brief_info`\u00A0— быстрая информация
> Нужна для\u00A0быстрого вывода данных из\u00A0других таблиц по\u00A0одному ID.

```sql
CREATE TABLE IF NOT EXISTS "brief_info" (
	"id" INTEGER NOT NULL,
	"id_participants" INTEGER,
	"id_accidents" INTEGER,
	PRIMARY KEY("id"),
	FOREIGN KEY ("id_participants") REFERENCES "participants"("id")
	ON UPDATE NO ACTION ON DELETE NO ACTION,
	FOREIGN KEY ("id_accidents") REFERENCES "accidents"("id")
	ON UPDATE NO ACTION ON DELETE NO ACTION,
);
```

## 3. Привязка 40\u00A0ручек к\u00A0таблицам БД

### 📦 КАТЕГОРИЯ А: Прямые данные (Direct Data)
*Простые SELECT/JOIN\u00A0— данные отдаются «как есть».*

#### Группа 1: Справочники и\u00A0метаданные

| # | Ручка | SQL-логика |
|---|---|---|
| 1 | `GET /dictionaries/regions` | `SELECT code, name FROM regions JOIN districts` |
| 2 | `GET /dictionaries/indicators` | статический ENUM из\u00A0конфига\u00A0+ `car_types` |
| 3 | `GET /dictionaries/weather` | `SELECT * FROM weather_types` |
| 4 | `GET /dictionaries/road-conditions` | `SELECT * FROM road_conditions` |
| 5 | `GET /dictionaries/accident-types` | `SELECT * FROM accident_types` |
| 6 | `GET /metadata/source-info` | `SELECT MAX(date_dtp), COUNT(*) FROM accidents` |

#### Группа 2: Детализированные списки

| # | Ручка | SQL-логика |
|---|---|---|
| 7 | `GET /accidents` | `SELECT ... FROM accidents LIMIT ? OFFSET ?` (все фильтры) |
| 8 | `GET /accidents/{id}` | `SELECT * FROM accidents WHERE id=?`\u00A0+ JOIN vehicles\u00A0+ participants |
| 9 | `GET /accidents/by-region/{reg_code}` | `WHERE region_code=?` |
| 10 | `GET /accidents/by-period` | `WHERE date_dtp BETWEEN ? AND ?` |
| 11 | `GET /accidents/by-indicator` | динамический WHERE по\u00A0переданному indicator |
| 12 | `GET /accidents/with-children` | `WHERE has_children=1` |
| 13 | `GET /accidents/drunk-drivers` | `WHERE has_drunk=1` |
| 14 | `GET /accidents/fled-drivers` | `WHERE driver_fled=1` |

#### Группа 3: Базовая статистика (Счётчики)

| # | Ручка | SQL-логика |
|---|---|---|
| 15 | `GET /stats/basic/total-count` | `SELECT COUNT(*) FROM accidents WHERE date_dtp BETWEEN ? AND ?` |
| 16 | `GET /stats/basic/total-pog` | `SELECT SUM(fatalities) FROM accidents WHERE ...` |
| 17 | `GET /stats/basic/total-ran` | `SELECT SUM(injured) FROM accidents WHERE ...` |
| 18 | `GET /stats/basic/by-district` | `GROUP BY d.id` через\u00A0JOIN regions → districts |
| 19 | `GET /stats/basic/populated-areas` | `GROUP BY is_city` |
| 20 | `GET /stats/basic/federal-roads` | `WHERE road_category='Федеральная'` |

### 🧮 КАТЕГОРИЯ Б: Расчётные показатели (Calculated Analytics)
*Агрегация, формулы, оконные функции\u00A0— вычисляется на\u00A0сервере.*

#### Группа 4: Тренды и\u00A0динамика

| # | Ручка | SQL-логика\u00A0+ формула |
|---|---|---|
| 21 | `GET /analytics/trends/monthly` | `GROUP BY strftime('%Y-%m', date_dtp)` → `(cur-prev)/prev*100\u00A0%` |
| 22 | `GET /analytics/trends/yearly` | `GROUP BY strftime('%Y', date_dtp)` → динамика год к\u00A0году |
| 23 | `GET /analytics/trends/prediction` | линейная регрессия по\u00A0monthly-ряду (Python `numpy.polyfit`) |
| 24 | `GET /analytics/seasonality` | `AVG(count) GROUP BY month` → коэффициент относительно среднего |

#### Группа 5: Сравнения и\u00A0рейтинги

| # | Ручка | SQL-логика\u00A0+ формула |
|---|---|---|
| 25 | `GET /analytics/compare/regions` | `WHERE region_code IN (?,?)` → side-by-side метрики |
| 26 | `GET /analytics/compare/periods` | текущий период\u00A0vs АППГ: `WHERE year IN (cur, cur-1)` |
| 27 | `GET /analytics/rating/danger-zones` | `GROUP BY road_name, region_code ORDER BY COUNT(*) DESC` |
| 28 | `GET /analytics/rating/safe-regions` | взвешенный индекс: `(COUNT*w1 + SUM(fatalities)*w2) / population` |

#### Группа 6: Сложные коэффициенты

| # | Ручка | Формула |
|---|---|---|
| 29 | `GET /analytics/metrics/severity-index` | `SUM(fatalities) / (SUM(fatalities)+SUM(injured))` |
| 30 | `GET /analytics/metrics/social-risk` | `SUM(fatalities) / population * 100\u00A0000` (JOIN regions) |
| 31 | `GET /analytics/metrics/transport-risk` | `SUM(fatalities) / vehicles * 10\u00A0000` (JOIN regions) |
| 32 | `GET /analytics/metrics/fatality-rate` | `SUM(fatalities) / (SUM(fatalities)+SUM(injured)) * 100` |
| 33 | `GET /analytics/metrics/children-trauma-rate` | `COUNT(WHERE has_children=1) / COUNT(*) * 100` |

#### Группа 7: Корреляционный анализ

| # | Ручка | SQL-логика\u00A0+ метод |
|---|---|---|
| 34 | `GET /analytics/correlation/weather-severity` | `GROUP BY weather_id` → AVG(fatalities+injured) per weather |
| 35 | `GET /analytics/correlation/experience-accidents` | `GROUP BY experience` (participants, role='Водитель') |
| 36 | `GET /analytics/correlation/lighting-time` | `GROUP BY lighting, hour_dtp` |
| 37 | `GET /analytics/correlation/road-defects` | `COUNT(WHERE has_road_defect=1) / COUNT(*) * 100` |

#### Группа 8: Сводные аналитические отчёты

| # | Ручка | Состав |
|---|---|---|
| 38 | `GET /analytics/reports/regional-dashboard` | severity_index\u00A0+ social_risk\u00A0+ transport_risk\u00A0+ trends |
| 39 | `GET /analytics/reports/participant-profile` | `GROUP BY role, gender, age_group, experience` → профиль виновника |
| 40 | `GET /analytics/reports/hotspots-map` | `GROUP BY round(coord_lat,3), round(coord_lon,3)` → кластеры |
