### Спецификация эндпоинтов для аналитики и статистики (Calculated Analytics)

В данном документе описаны эндпоинты, возвращающие результат математической обработки данных. Предназначены для визуализации трендов, сравнения регионов и прогнозирования ситуации на дорогах.

---

#### 1. Группа: Общая и региональная статистика

**GET /stats/summary**
Общий "пульс" системы за выбранный период.
*   **Параметры:** `date_from`, `date_to`.
*   **Логика:** Выполняет три агрегации в одном запросе: `COUNT(*)` (всего ДТП), `SUM(fatalities)` (погибшие), `SUM(injured)` (раненые).
*   **Ответ:** Объект со сводными счетчиками для дашборда.

**GET /stats/regions**
Рейтинговая таблица регионов.
*   **Параметры:** `sort_by` (accidents/fatalities), `order` (asc/desc).
*   **Логика:** `SELECT region_code, name, COUNT(*), SUM(fatalities) FROM accidents GROUP BY region_code`. Применяется `ORDER BY` на уровне БД для эффективной сортировки больших наборов данных.

**GET /stats/regions/{code}**
Паспорт безопасности конкретного региона.
*   **Логика:** Помимо базовой агрегации, вычисляет **Severity Index (Индекс тяжести)** прямо в Python-сервисе: `pog / (pog + ran)`.
*   **Ответ:** Объект с абсолютными цифрами и вычисленным коэффициентом тяжести.

**GET /stats/timeline**
Данные для линейных графиков динамики.
*   **Параметры:** `interval` (day/month/year).
*   **Логика:** SQL-группировка с использованием функции `strftime`. При `interval=month` логика: `GROUP BY strftime('%Y-%m', date_dtp)`. Позволяет увидеть сезонные колебания.

#### 2. Группа: Тренды и Динамика (Trends)

**GET /analytics/trends/monthly**
Анализ темпов роста или снижения аварийности.
*   **Логика:** Вычисляется разница между текущим и предыдущим месяцем. Формула: `((V_cur - V_prev) / V_cur) * 100%`.
*   **Назначение:** Позволяет понять, эффективно ли работают меры безопасности в текущем месяце относительно предыдущего.

**GET /analytics/trends/yearly**
Глобальный анализ изменений по годам.
*   **Логика:** Агрегация данных в разрезе лет. Позволяет строить гистограммы для годовых отчетов.

**GET /analytics/trends/prediction**
Математический прогноз на будущий период.
*   **Логика:** Используется библиотека `numpy` (функция `polyfit`). На основе временного ряда за последние 12 месяцев строится линейная регрессия, предсказывающая значение на месяц `N+1`.
*   **Ответ:** Предполагаемое количество ДТП и доверительный интервал (погрешность).

**GET /analytics/seasonality**
Выявление влияния времен года на безопасность.
*   **Логика:** Данные разбиваются на 4 сезона (Зима: 12,1,2; Весна: 3,4,5 и т.д.). Вычисляется среднее количество ДТП для каждого сезона и сравнивается с общим среднегодовым значением.

#### 3. Группа: Сравнения и Рейтинги (Comparison)

**GET /analytics/compare/regions**
Инструмент для сравнительного анализа двух субъектов.
*   **Параметры:** `codes=77,50`.
*   **Логика:** Параллельная выборка данных для указанных кодов и расчет удельных показателей (на 100к населения) для корректного сравнения маленьких и больших регионов.

**GET /analytics/compare/periods**
Сравнение с "аналогичным периодом прошлого года" (АППГ).
*   **Логика:** Берется текущий диапазон дат и такой же диапазон ровно год назад. Вычисляется дельта (разница) в процентах по всем ключевым показателям.

**GET /analytics/rating/danger-zones**
Поиск "очагов аварийности" (черных точек).
*   **Логика:** Группировка по `road_name` и `locality` с сортировкой по убыванию `COUNT(*)`. Выводит топ-10 самых опасных улиц или участков трасс.

**GET /analytics/rating/safe-regions**
Формирование рейтинга безопасности.
*   **Логика:** Сложная формула ранжирования: `Safety_Score = (Accidents_per_100k * 0.4) + (Severity_Index * 0.6)`. Чем ниже индекс, тем выше место региона в рейтинге.

#### 4. Группа: Сложные коэффициенты (Metrics)

**GET /analytics/metrics/severity-index**
Расчет индекса тяжести последствий.
*   **Формула:** `SUM(fatalities) / (SUM(fatalities) + SUM(injured))`. Показывает вероятность летального исхода при попадании в ДТП.

**GET /analytics/metrics/social-risk**
Социальный риск (по методике ГИБДД).
*   **Формула:** `(SUM(fatalities) / Population) * 100,000`. Используются данные о населении из таблицы `regions`.

**GET /analytics/metrics/transport-risk**
Транспортный риск.
*   **Формула:** `(SUM(fatalities) / Registered_Vehicles) * 10,000`. Показывает смертность относительно уровня автомобилизации региона.

**GET /analytics/metrics/children-trauma-rate**
Удельный вес детского травматизма.
*   **Логика:** `(COUNT(where has_children=1) / COUNT(*)) * 100`.

---

#### 5. Группа: Корреляционный анализ (Correlation)

**GET /analytics/correlation/weather-severity**
Влияние погоды на тяжесть ДТП.
*   **Логика:** Сравнивается средний `Severity Index` для категорий "Дождь/Туман" и "Ясно".

**GET /analytics/correlation/experience-accidents**
Зависимость аварийности от стажа.
*   **Логика:** Группировка виновников (participants.is_culprit=1) по стажу (0-2 года, 2-5 лет, 5-10 лет, 10+). Выявляет самые опасные категории водителей.

**GET /analytics/correlation/lighting-time**
Анализ освещенности.
*   **Логика:** Группировка по полю `lighting`. Сравнение аварийности в светлое время суток и в темное (с учетом наличия/отсутствия фонарей).

---

#### 6. Группа: Сводные отчеты (Reports)

**GET /analytics/reports/regional-dashboard**
Агрегатор для фронтенд-дашборда.
*   **Логика:** Вызывает внутренние методы всех аналитических функций для одного региона и упаковывает результат в один массивный JSON.

**GET /analytics/reports/participant-profile**
"Портрет типичного виновника".
*   **Логика:** Статистическое определение моды (самого частого значения) для полей: пол, возраст, стаж, состояние опьянения среди виновников ДТП.

**GET /analytics/reports/hotspots-map**
Данные для отрисовки тепловой карты.
*   **Логика:** Кластеризация координат `coord_lat`, `coord_lon` с округлением до 3 знаков после запятой. Группировка близких точек для передачи фронтенду плотности ДТП.
