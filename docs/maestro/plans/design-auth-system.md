# Дизайн-документ: Система управления API-ключами (Student Auth System)

## 1. Концепция
Система позволяет администратору (преподавателю) выдавать именные ключи студентам. Доступ к основным методам аналитики возможен только при наличии активного ключа в заголовке `X-API-KEY`.

## 2. Схема Базы Данных (`APIKey`)
- `id`: INTEGER (PK)
- `owner_name`: VARCHAR(255) (Имя студента/группы)
- `key_hash`: VARCHAR(64) (SHA-256 хеш ключа)
- `prefix`: VARCHAR(8) (Первые символы ключа для идентификации в логах)
- `is_active`: BOOLEAN (Возможность быстрого отзыва доступа)
- `created_at`: DATETIME

## 3. Механизм безопасности
- **Генерация**: Используется `secrets.token_urlsafe(32)`.
- **Хранение**: В БД записывается только хеш. Прямой текст ключа показывается пользователю **только один раз** при создании.
- **Мастер-ключ**: Операция генерации новых ключей (`POST /auth/generate`) защищена глобальным `MASTER_KEY` из окружения.

## 4. API Эндпоинты
- `POST /api/v1/auth/keys`: Создать новый ключ (требует Master-Key).
- `GET /api/v1/auth/me`: Показать информацию о текущем владельце ключа.
- `GET /api/v1/auth/keys`: Список всех выданных ключей (только метаданные).

## 5. Интеграция
Функция `verify_api_key` в `main.py` заменяется на асинхронную проверку по таблице `api_keys`.
