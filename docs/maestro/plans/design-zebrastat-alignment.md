# Дизайн-документ: Синхронизация API и Спецификации «Зебра-стат»

## 1. Цель
Обеспечить 100% соответствие программной реализации и документации в `docs/`. Устранить «архитектурный долг» в виде нереализованных, но задекларированных функций.

## 2. Ключевые изменения

### Справочники
- Внедрение эндпоинта `GET /dictionaries/districts`.
- Обертывание всех ответов справочников в структуру `{ "total": int, "data": [] }` согласно `Наборы данных.md`.

### Группировка API
- Разделение текущего `analytics.router` на два логических блока:
    - **`/stats/`**: Базовая агрегация (summary, regions, timeline).
    - **`/analytics/`**: Сложная математика (trends, prediction, seasonality, ratings, correlations, compare, reports).

### Новые модули аналитики
- **Сравнение (Comparison)**: Реализация логики удельных показателей (на 100к населения) для честного сравнения регионов.
- **Метрики (Metrics)**: Выделение расчета `severity-index`, `social-risk`, `transport-risk` и `children-trauma-rate` в отдельные эндпоинты.
- **Тепловые карты (Hotspots)**: Реализация кластеризации координат на стороне Python (округление до 3 знаков).

## 3. Технологические ограничения
- Продолжение использования **Pure Python** (стандартная библиотека) для всех расчетов.
- Сохранение полной асинхронности через `AsyncSession`.
