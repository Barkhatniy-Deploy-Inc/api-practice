# Design Document: Backend Bugfixes and Refactoring

## 1. Problem Statement
**Цель:** Устранить критические недостатки архитектуры и безопасности, выявленные в ходе аудита.
**Контекст:** Проект находится на стадии скелета, требуются правки в слоях DB, CRUD, API и Core.

## 2. Approach
### Database Session
- Удаление автоматического `commit()` из `get_db` для повышения производительности и корректности.

### CRUD Refactoring (Base CRUD Class)
- Создание базового класса для инкапсуляции логики фильтрации (DRY).

### Localization
- Переход на маппинг по типам ошибок (`error.type`) для стабильного перевода на русский язык.

### Service Layer
- Полная инкапсуляция бизнес-логики (Thin Controllers).

## 3. Risk Assessment
- **Изменение API**: Риск минимален, так как контракты эндпоинтов (схемы) сохраняются.
- **Математика**: Повторная проверка формул после переноса в сервисы через существующие тесты.

## 4. Success Criteria
- Отсутствие дублирования кода в `crud_accidents.py`.
- Корректная работа локализованных ошибок.
- Прохождение всех тестов в `tests/`.
