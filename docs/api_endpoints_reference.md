# Справочник эндпоинтов API (API Endpoints Reference)

В данном документе приведено детальное описание всех доступных методов DTP Analytics API с примерами использования.

---

## 1. Группа: Справочники (Dictionaries)
*Все запросы требуют заголовок X-API-KEY.*

### `GET /api/v1/dictionaries/regions`
*   **Описание:** Список всех субъектов РФ.
*   **Результат:** Массив объектов `[{"id": "77", "name": "гор. Москва"}, ...]`.
*   **Зачем:** Для построения выпадающих списков фильтрации на фронтенде.

### `GET /api/v1/dictionaries/districts`
*   **Описание:** Список федеральных округов.
*   **Результат:** `[{"id": 1, "code": "CFO", "name": "Центральный федеральный округ"}]`.

---

## 2. Группа: Реестр ДТП (Accidents)

### `GET /api/v1/accidents/`
*   **Описание:** Получение списка происшествий с фильтрацией.
*   **Параметры:**
    *   `limit` (int): Кол-во записей (default: 50).
    *   `region_code` (str): Код региона (например, "1114").
    *   `has_children` (bool): Только ДТП с детьми.
*   **Пример:** `/api/v1/accidents/?has_children=true&limit=10`

### `GET /api/v1/accidents/{id}`
*   **Описание:** Полная "карточка" ДТП.
*   **Результат:** Объект с данными о месте, погоде, всех ТС и участниках (включая их роли и виновность).

---

## 3. Группа: Аналитика (Analytics)

### `GET /api/v1/analytics/trends/monthly`
*   **Описание:** Сравнение текущего месяца с предыдущим.
*   **Результат:** Объект с `growth_rate` (темп роста). Позволяет понять, становится ли на дорогах безопаснее.

### `GET /api/v1/analytics/predictions/accidents`
*   **Описание:** Прогноз количества ДТП на следующий месяц.
*   **Логика:** Линейная регрессия на основе исторических данных.
*   **Результат:** `{"predicted_value": X, "confidence_interval": Y}`.

---

## 4. Группа: Системные (System)

### `GET /health`
*   **Описание:** Проверка доступности сервиса.
*   **Авторизация:** НЕ ТРЕБУЕТСЯ.
*   **Зачем:** Для мониторинга работоспособности сервера.
