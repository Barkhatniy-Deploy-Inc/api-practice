# Спецификация системы токенов и безопасности (Token System Design)

В данном документе описывается архитектура системы доступа, управления API-ключами и механизмов авторизации для DTP Analytics API.

---

## 1. Концепция доступа
Доступ к API осуществляется на основе статических API-ключей (X-API-KEY), передаваемых в заголовках HTTP-запросов. Система спроектирована с учетом простоты для студентов и высокой безопасности для серверной части.

### Технические решения:
*   **Место хранения:** Переменные окружения (`.env` файл).
*   **Тип сравнения:** Constant Time Comparison (защита от атак по времени).
*   **Область действия:** Глобальная (защищает все эндпоинты API).

---

## 2. Формат токенов
Токен представляет собой строку в формате `sk_dtp_practice_<random_hex>`.
*   **Длина:** Минимум 32 символа.
*   **Пример:** `sk_dtp_practice_secure_key_12345678`

---

## 3. Механизм проверки (Security Layer)
Реализуется на уровне Middleware или Dependencies в FastAPI.

### Логика работы:
1. Перехват заголовка `X-API-KEY`.
2. Если заголовок отсутствует -> Ошибка `403 Forbidden` ("Ключ доступа не предоставлен").
3. Если ключ не совпадает со значением из настроек -> Ошибка `403 Forbidden` ("Некорректный ключ доступа").
4. Все попытки доступа с неверным ключом логируются (без вывода самого ключа в логи).

---

## 4. Группы доступа (Будущее расширение)
На текущем этапе реализована единая роль "Developer". В будущем возможна сегментация:
*   `Read-Only`: Доступ только к GET-запросам (справочники, аналитика).
*   `Admin`: Доступ к загрузке данных (POST /sync).

---

## 5. Инструкция для студентов
Для выполнения запросов к API необходимо добавить заголовок в клиент:
```javascript
headers = {
  "X-API-KEY": "ваш_ключ_из_файла_.env"
}
```
