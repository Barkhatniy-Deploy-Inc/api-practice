# Цели и задачи проекта: DTP Analytics API

Проект направлен на создание профессионального API для анализа данных о дорожно-транспортных происшествиях и обучения методам проектирования и потребления сетевых ресурсов.

---

## 🔍 Описание предметной области

Предметной областью проекта является **статистика и анализ безопасности дорожного движения (БДД)**. 

### Основные аспекты анализа:
*   **Географический фактор:** Распределение аварийности по регионам, типам населенных пунктов и конкретным участкам дорожной сети.
*   **Фактор времени:** Выявление пиков аварийности по часам суток, дням недели и сезонным изменениям (месяцы).
*   **Человеческий фактор:** Влияние стажа вождения, состояния водителя (алкогольное опьянение) и соблюдения скоростного режима.
*   **Техногенный и природный факторы:** Зависимость безопасности от погодных условий (туман, гололед), освещенности и состояния дорожного полотна.
*   **Тяжесть последствий:** Количественные показатели погибших и раненых в зависимости от типа ДТП (столкновение, наезд на пешехода и т.д.).

---

## 🎯 Глобальные цели проекта

1.  **Инженерная (Проектирование):** Получение практического опыта в проектировании сложной архитектуры RESTful API (40+ эндпоинтов) с использованием современного стека (FastAPI).
2.  **Образовательная (Потребление):** Обучение группы студентов методам работы с защищенными API-запросами, обработке сложных JSON-структур и их визуализации.

---

## 🛠 Задачи проекта

### 1. Задачи проектирования (Архитектура)
*   **Разработка модели данных:** Создание JSON-схемы, охватывающей все аспекты предметной области (от метеоусловий до стажа водителя).
*   **Документирование системы:** Обеспечение 100% покрытия эндпоинтов интерактивной документацией Swagger для внешних разработчиков.
*   **Система аудита:** Настройка логирования каждого запроса для анализа нагрузки и безопасности доступа.

### 2. Задачи реализации (Backend)
*   **Инфраструктура:** Реализация ядра сервера и Middleware для контроля доступа (`X-API-KEY`) и ведения журналов событий (`logs`).
*   **Бизнес-логика:** Написание 40 уникальных маршрутов для предоставления различных срезов данных.

### 3. Задачи для группы студентов (Потребители)
*   **Взаимодействие:** Отработка навыков выполнения защищенных HTTP-запросов и обработки ошибок.
*   **Аналитика:** Трансформация полученных JSON-данных в интерактивные отчеты и графики.
