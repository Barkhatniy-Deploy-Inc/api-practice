# Цели и задачи проекта: DTP Analytics API

Проект имеет двойную образовательную ценность: отработка навыков системного проектирования и планирования со стороны команды разработчиков, и обучение методам взаимодействия с API со стороны группы студентов.

---

## 🎯 Глобальные цели

### Для команды планирования (Рабы):
1.  **Проектирование систем:** Получение практического опыта в составлении технического задания для сложной системы (40+ эндпоинтов).
2.  **Техническое документирование:** Создание исчерпывающей документации, по которой сторонние разработчики смогут работать без дополнительных консультаций.
3.  **Архитектурное планирование:** Разработка логичной структуры ресурсов, системы безопасности и аудита (логирования).

### Для группы студентов (Нахлебники):
1.  **Работа с API:** Обучение выполнению HTTP-запросов, работе с заголовками (Headers) и обработке различных статус-кодов.
2.  **Анализ данных:** Получение навыков разбора сложных JSON-структур и их трансформации для визуализации.

---

## 🛠 Задачи проекта

### 1. Задачи проектирования (Рабы на галере)
*   **Составление Спецификации:** Описание 40 эндпоинтов с четким определением входных параметров и форматов ответа.
*   **Проектирование системы логирования:** Разработка структуры логов, позволяющей отслеживать действия каждого студента (аудит запросов).
*   **Определение модели данных:** Создание эталонного JSON-объекта, который покроет все аналитические потребности проекта.
*   **Выбор стека:** Обоснование выбора технологий (FastAPI) с точки зрения удобства документирования (Swagger).

### 2. Задачи реализации API (Backend (задник))
*   **Фундамент:** Реализация ядра сервера и механизмов Middleware (логирование + безопасность).
*   **Наполнение:** Создание логики 40 эндпоинтов, предоставляющих доступ к различным срезам статистики ДТП.
*   **Публикация документации:** Настройка автоматического Swagger-интерфейса для «бесшовного» тестирования студентами.

### 3. Задачи обучения студентов (Группа нахлебников)
*   **Аутентификация:** Научиться передавать `X-API-KEY` и обрабатывать ошибку `401 Unauthorized`.
*   **Исследование:** Самостоятельно изучить Swagger-документацию и найти нужные ручки для решения конкретных задач (например, «найти все ДТП в дождь»).
*   **Визуализация:** На основе полученных данных построить интерактивные графики и карты.

---

## 📈 Критерии качества (если есть)
1.  **Полнота документации:** Студент может понять принцип работы любой из 40 ручек без обращения к преподавателю.
2.  **Прослеживаемость (Traceability):** Файл логов позволяет точно сказать, кто, когда и какой запрос выполнил.
3.  **Валидность:** Все эндпоинты возвращают данные, строго соответствующие описанным Pydantic-схемам.
