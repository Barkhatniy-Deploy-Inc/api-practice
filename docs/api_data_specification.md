### Спецификация эндпоинтов для выдачи данных из БД (Data Delivery)

Этот документ содержит детальное описание эндпоинтов, предназначенных для получения справочной информации и реестра происшествий. Данные извлекаются напрямую из SQLite БД с применением фильтрации и объединения таблиц.

---

#### 1. Группа: Справочники и метаданные (Dictionaries)

**GET /dictionaries/regions**
Возвращает полный перечень субъектов Российской Федерации с их характеристиками.
*   **Логика:** Выполняется `SELECT` из таблицы `regions`. Для получения названия федерального округа применяется `LEFT JOIN` с таблицей `districts` по ключу `district_id`.
*   **Возвращаемые поля:** Код региона (ISO/ГИБДД), полное наименование, название округа, численность населения (для расчетов рисков) и количество зарегистрированных ТС.
*   **Пример ответа:**
```json
[
  {
    "region_code": "77",
    "region_name": "г. Москва",
    "district_name": "Центральный федеральный округ",
    "population": 12635466,
    "vehicles_count": 5000000
  }
]
```

**GET /dictionaries/indicators**
Список классификаторов (индикаторов), по которым можно фильтровать статистику.
*   **Логика:** Возвращает статический массив объектов, определенных в ТЗ (drunk, children, fled и т.д.). Используется фронтендом для построения выпадающих списков фильтрации.
*   **Ответ:** Список пар `id` (технический ключ) и `name` (человекочитаемое название).

**GET /dictionaries/weather**
Справочник погодных условий на момент ДТП.
*   **Логика:** Простая выборка `SELECT id, name FROM weather_types`.
*   **Назначение:** Позволяет сопоставить `weather_id` из таблицы `accidents` с текстовым описанием ("Ясно", "Дождь", "Снегопад").

**GET /dictionaries/road-conditions**
Классификатор состояния дорожного полотна.
*   **Логика:** Выборка из `road_conditions`. Описывает физическое состояние покрытия: "Сухое", "Мокрое", "Гололедица".

**GET /dictionaries/accident-types**
Официальный перечень видов дорожно-транспортных происшествий.
*   **Логика:** Выборка из `accident_types`. Включает такие виды как: "Столкновение", "Наезд на пешехода", "Опрокидывание".

#### 2. Группа: Детализированные списки (Accident Registry)

**GET /accidents**
Основной реестр происшествий с мощной системой фильтрации и пагинации.
*   **Параметры:**
    *   `limit` (default 50): количество записей.
    *   `offset`: смещение для постраничного вывода.
    *   `region_code`: фильтр по субъекту.
    *   `date_from`/`date_to`: временной диапазон.
    *   `has_children` (boolean): только ДТП с детьми.
    *   `has_drunk` (boolean): только ДТП с нетрезвыми водителями.
*   **Логика:** Динамическое построение SQL-запроса к таблице `accidents`. При наличии флагов `has_children` или `has_drunk` добавляются соответствующие условия `WHERE`.
*   **Ответ:** Объект с общим количеством найденных записей (`total`) и массивом кратких карточек ДТП.

**GET /accidents/{id}**
Максимально полная информация по конкретному инциденту.
*   **Логика:** Агрегация данных из четырех источников:
    1. Базовая инфо из `accidents` (время, координаты, адрес).
    2. Погода и дорога через JOIN со справочниками.
    3. Список транспортных средств из `vehicles` (марка, модель, год).
    4. Список всех участников из `participants`, связанных с этим ДТП и ТС (роль, возраст, стаж, состояние здоровья).
*   **Ответ:** Глубоко вложенный JSON-объект, описывающий "сцену" происшествия целиком.

**GET /accidents/by-region/{reg_code}**
Быстрая выборка происшествий для конкретного региона.
*   **Логика:** Упрощенная версия `/accidents`, где `WHERE region_code = :reg_code` является обязательным условием.

**GET /accidents/by-period**
Архив ДТП за выбранный промежуток времени.
*   **Логика:** Фильтрация по полю `date_dtp` с использованием оператора `BETWEEN`. Ожидает параметры в формате `YYYY-MM-DD`.

**GET /accidents/with-children**
Специализированный эндпоинт для анализа детского травматизма.
*   **Логика:** `SELECT * FROM accidents WHERE has_children = 1`. Логика опирается на денормализованный флаг в главной таблице для высокой скорости ответа.

**GET /accidents/drunk-drivers**
Выборка инцидентов с нетрезвыми участниками.
*   **Логика:** Поиск в таблице `accidents` записей, где `has_drunk = 1`.

**GET /accidents/fled-drivers**
Реестр ДТП, где виновник скрылся с места происшествия.
*   **Логика:** `SELECT * FROM accidents WHERE driver_fled = 1`.

#### 3. Группа: Базовая статистика (Basic Stats)

**GET /stats/basic/total-count**
Счетчик общего количества происшествий.
*   **Логика:** Выполняет `SELECT COUNT(*)` с учетом переданных фильтров по дате и региону. Позволяет быстро получить цифру для заголовка отчета.

**GET /stats/basic/total-pog**
Количество погибших (летальные исходы).
*   **Логика:** `SELECT SUM(fatalities) FROM accidents`. Суммирует значение поля `fatalities` за период.

**GET /stats/basic/total-ran**
Количество раненых (пострадавших).
*   **Логика:** `SELECT SUM(injured) FROM accidents`. Агрегация поля `injured`.

**GET /stats/basic/by-district**
Распределение показателей по федеральным округам.
*   **Логика:** Тройной JOIN: `accidents` -> `regions` -> `districts`. Группировка `GROUP BY districts.name`. Считает количество ДТП и сумму пострадавших для каждого округа.

**GET /stats/basic/populated-areas**
Сравнение безопасности в городах и на трассах.
*   **Логика:** Группировка по флагу `is_city`. Позволяет понять, где происходит больше аварий: в плотной городской застройке или на скоростных магистралях.

**GET /stats/basic/federal-roads**
Анализ аварийности на ключевых транспортных артериях.
*   **Логика:** Фильтрация `WHERE road_category = 'Федеральная'`. Группировка по полю `road_name` для выявления самых опасных трасс.

---

#### 4. Группа: Метаданные (Metadata)

**GET /metadata/source-info**
Служебная информация о наборе данных.
*   **Логика:** Чтение из таблицы `api_metadata`. Возвращает дату последнего импорта данных из ГИБДД, общее кол-во строк в БД и ссылку на первоисточник.
