# Техническое задание: DTP Analytics API (Python + FastAPI)

## 1. Общее описание
Проект представляет собой API для работы со статистикой ДТП в образовательных целях. 
**Стек:** Python 3.10+, FastAPI, Pydantic, Uvicorn.

---

## 2. Архитектурные требования
*   **Авто-документация:** Доступна по адресу `/docs` (Swagger UI).
*   **Безопасность:** Проверка заголовка `X-API-KEY`.
*   **Логирование:** Middleware, записывающий каждый запрос в файл `logs/api_access.log`.
*   **Валидация:** Строгая типизация всех входящих и исходящих данных через Pydantic-схемы.

## 5. Спецификация данных

### 5.1. Входные данные (Query Parameters)
Основано на реальных фильтрах системы stat.gibdd.ru.
Для GET-запросов коллекции `/accidents` доступны параметры:
*   **limit** / **offset**: Пагинация (по умолчанию 100 / 0).
*   **period_start** / **period_end** (YYYY-MM): Фильтрация по месяцам.
*   **region_code** (string): Код региона (например, "77" для Москвы).
*   **district_code** (string): Код федерального округа (ЦФО, ЮФО).
*   **indicator** (enum): Специфический показатель:
    *   `pedestrians` (с участием пешеходов)
    *   `children_under_16` (дети до 16 лет)
    *   `oncoming_lane` (выезд на встречную полосу)
    *   `driver_fled` (водитель скрылся)
*   **format** (enum: `json`, `csv`): Формат выдачи данных.

### 5.2. Структура JSON-ответа (Карточка ДТП)
Модель данных полностью соответствует официальной спецификации ГИБДД (opendataapi/v1/kartdtp).

### 5.3. Описание ключевых полей
*   **empt_number**: Уникальный системный номер ДТП.
*   **dtpv**: Вид дорожно-транспортного происшествия.
*   **coord_w / coord_l**: Географические координаты (широта/долгота).
*   **k_ts / k_uch**: Количество транспортных средств и участников соответственно.
*   **spog / s_pch / osv**: Массивы условий (погода, состояние покрытия, освещенность).
*   **v_st**: Водительский стаж участника (в годах).
*   **alco**: Сведения о состоянии опьянения участника.

## 6. Группы ручек (Всего: 40)
*Список эндпоинтов адаптирован под реальные показатели ГИБДД.*

### 📂 Группа 1: Базовые данные и Справочники (5 ручек)
1.  `GET /accidents` — Список всех ДТП (с фильтрами и пагинацией).
2.  `GET /accidents/{id}` — Детальная карточка конкретного происшествия.
3.  `GET /dictionaries/regions` — Справочник регионов и ФО РФ.
4.  `GET /dictionaries/indicators` — Справочник показателей (выезд на встречку, пешеходы и т.д.).
5.  `GET /dictionaries/weather` — Типы погодных условий.

### 🗺 Группа 2: Территориальная аналитика (8 ручек)
6.  `GET /stats/territory/rf` — Сводка по Российской Федерации.
7.  `GET /stats/territory/districts` — Сравнение Федеральных округов.
8.  `GET /stats/territory/regions/top` — Топ самых аварийных регионов.
9.  `GET /stats/territory/{region_code}/summary` — Сводка по конкретному региону.
10. `GET /stats/territory/populated-areas` — Аварийность в населенных пунктах.
11. `GET /stats/territory/highways` — Аварийность на трассах вне населенных пунктов.
12. `GET /stats/territory/federal-roads` — ДТП на федеральных автодорогах (ФАД).
13. `GET /stats/territory/railway-crossings` — ДТП на железнодорожных переездах.

### 📅 Группа 3: Временные ряды (7 ручек)
14. `GET /stats/time/years` — Динамика по годам (последние 10 лет).
15. `GET /stats/time/months` — Сезонная статистика (распределение по месяцам).
16. `GET /stats/time/days-of-week` — Анализ аварийности по дням недели.
17. `GET /stats/time/hours` — Почасовое распределение (выявление часов пик).
18. `GET /stats/time/weekends-vs-weekdays` — Сравнение будней и выходных.
19. `GET /stats/time/holidays` — Статистика в праздничные дни.
20. `GET /stats/time/night-time` — Анализ ДТП в темное время суток.

### 👥 Группа 4: Участники и Транспорт (8 ручек)
21. `GET /stats/participants/pedestrians` — ДТП с наездом на пешеходов.
22. `GET /stats/participants/children` — Детский дорожно-транспортный травматизм.
23. `GET /stats/participants/cyclists` — ДТП с участием велосипедистов и СИМ.
24. `GET /stats/participants/public-transport` — Аварии с участием ОТ.
25. `GET /stats/participants/experience-stats` — Зависимость ДТП от стажа водителя.
26. `GET /stats/participants/age-groups` — Возрастные группы виновников.
27. `GET /stats/participants/gender` — Гендерная статистика происшествий.
28. `GET /stats/transport/types` — Распределение по видам ТС (легковые, грузовые, мото).

### 🌦 Группа 5: Внешние факторы и Причины (7 ручек)
29. `GET /stats/factors/weather` — Влияние погодных условий (осадки, туман).
30. `GET /stats/factors/road-conditions` — Влияние состояния дорожного полотна.
31. `GET /stats/factors/lighting` — Уровень освещенности участка дороги.
32. `GET /stats/causes/alcohol` — ДТП по вине водителей в состоянии опьянения.
33. `GET /stats/causes/speeding` — Нарушения скоростного режима.
34. `GET /stats/causes/road-defects` — ДТП из-за неудовлетворительного состояния дорог.
35. `GET /stats/factors/infrastructure` — Роль светофоров, знаков и разметки.

### 🏥 Группа 6: Последствия и Тяжесть (5 ручек)
36. `GET /stats/severity/fatalities` — Динамика смертности в ДТП.
37. `GET /stats/severity/injuries` — Статистика травмированных.
38. `GET /stats/severity/index` — Расчет индекса тяжести последствий.
39. `GET /stats/severity/first-aid` — Эффективность оказания первой помощи на месте.
40. `GET /stats/severity/rescue-operations` — Время прибытия экстренных служб.
