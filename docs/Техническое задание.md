# Техническое задание: DTP Analytics API (Python + FastAPI)

## 1. Общее описание
Проект представляет собой Enterprise-grade API (профессиональный программный интерфейс) для работы со статистикой ДТП. 
**Стек (набор технологий):** Python 3.11+, FastAPI (фреймворк для API), Pydantic (валидация данных), Uvicorn (сервер).

---

## 2. Архитектурные требования
*   **Авто-документация:** Доступна по адресу `/docs` (интерфейс Swagger UI для живого тестирования).
*   **Безопасность:** Проверка заголовка `X-API-KEY` (статический ключ доступа).
*   **Логирование:** Middleware (промежуточный слой кода), записывающий каждый запрос в файл `logs/api_access.log`.
*   **Валидация (проверка):** Строгая типизация данных через Pydantic-схемы (модели данных).

---

## 5. Спецификация данных

### 5.1. Входные данные (Query Parameters - параметры в URL)
Основано на реальных фильтрах системы stat.gibdd.ru.
Для GET-запросов (получение данных) коллекции `/accidents` доступны параметры:
*   **limit** / **offset**: Pagination (постраничный вывод данных).
*   **period_start** / **period_end** (YYYY-MM): Фильтрация по периоду (месяцам).
*   **region_code** (string): Код субъекта (например, "77" для Москвы).
*   **district_code** (string): Код федерального округа (ЦФО, ЮФО).
*   **indicator** (enum - список значений): Специфический показатель (например, `driver_fled` - водитель скрылся).
*   **format** (enum: `json`, `csv`): Format (способ представления данных).

### 5.2. Структура JSON-ответа (Карточка ДТП)
Модель данных полностью соответствует официальной спецификации ГИБДД (opendataapi/v1/kartdtp).

```json
{
  "empt_number": "140000008",        // Уникальный номер ДТП
  "dtpv": "Наезд на пешехода",       // Вид происшествия
  "coord_w": "50.406494",            // Широта (гео-координата)
  "coord_l": "36.833982",            // Долгота (гео-координата)
  "location": {
    "district": "ШЕБЕКИНСКИЙ РАЙОН",
    "np": "Шебекино",
    "dor": "а/д Белгород-Шебекино"
  },
  "outcome": { "pog": 0, "ran": 1 }, // Погибшие и раненые
  "ts_info": [                       // Транспортные средства
    {
      "marka_ts": "RENAULT",
      "ts_uch": [                    // Участники ДТП
        {
          "kt_uch": "Водитель",
          "alco": "Нет опьянения",
          "v_st": 12                 // Стаж вождения
        }
      ]
    }
  ]
}
```

## 6. Реестр эндпоинтов (Всего: 40)

Эндпоинты (конечные точки API) разделены на базовые данные и сложные аналитические показатели (математика).

---

### 📦 КАТЕГОРИЯ А: Прямые данные (Direct Data)
*Данные, предоставляемые источником (ГИБДД/Росстат) в сыром или базовом виде.*

#### Группа 1: Справочники и метаданные
1.  `GET /dictionaries/regions` — Список субъектов РФ.
2.  `GET /dictionaries/indicators` — Классификатор показателей аварийности.
3.  `GET /dictionaries/weather` — Виды погодных условий.
4.  `GET /dictionaries/road-conditions` — Состояние дорожного покрытия.
5.  `GET /dictionaries/accident-types` — Виды происшествий.
6.  `GET /metadata/source-info` — Сведения о дате обновления и объеме данных.

#### Группа 2: Детализированные списки
7.  `GET /accidents` — Полный список ДТП (с фильтрами).
8.  `GET /accidents/{id}` — Полная карточка конкретного ДТП.
9.  `GET /accidents/by-region/{reg_code}` — Выборка ДТП по конкретному региону.
10. `GET /accidents/by-period` — Выборка ДТП за указанный промежуток времени.
11. `GET /accidents/by-indicator` — ДТП, отфильтрованные по одному показателю.
12. `GET /accidents/with-children` — Выборка всех ДТП с участием несовершеннолетних.
13. `GET /accidents/drunk-drivers` — Выборка ДТП с участием водителей в состоянии опьянения.
14. `GET /accidents/fled-drivers` — ДТП, с места которых водитель скрылся.

#### Группа 3: Базовая статистика (Счетчики)
15. `GET /stats/basic/total-count` — Общее количество ДТП за период.
16. `GET /stats/basic/total-pog` — Общее количество погибших.
17. `GET /stats/basic/total-ran` — Общее количество раненых.
18. `GET /stats/basic/by-district` — Общие цифры в разрезе федеральных округов.
19. `GET /stats/basic/populated-areas` — Количество ДТП в городах vs сельская местность.
20. `GET /stats/basic/federal-roads` — Количество ДТП на федеральных трассах.

---

### 🧮 КАТЕГОРИЯ Б: Расчетные показатели (Calculated Analytics)
*Показатели, получаемые путем математических вычислений, агрегации и анализа на стороне сервера.*

#### Группа 4: Тренды и Динамика
21. `GET /analytics/trends/monthly` — Процентное изменение аварийности месяц к месяцу.
22. `GET /analytics/trends/yearly` — Динамика изменений показателей по годам.
23. `GET /analytics/trends/prediction` — Прогноз аварийности на следующий месяц (линейная экстраполяция).
24. `GET /analytics/seasonality` — Коэффициент сезонности (влияние времен года на аварийность).

#### Группа 5: Сравнения и Рейтинги
25. `GET /analytics/compare/regions` — Сравнение показателей двух и более регионов.
26. `GET /analytics/compare/periods` — Сравнение текущего периода с АППГ (аналогичный период прошлого года).
27. `GET /analytics/rating/danger-zones` — Рейтинг самых опасных участков дорог.
28. `GET /analytics/rating/safe-regions` — Индекс безопасности регионов (взвешенный показатель).

#### Группа 6: Сложные коэффициенты и Математика
29. `GET /analytics/metrics/severity-index` — Расчет индекса тяжести последствий (погибшие / (погибшие + раненые)).
30. `GET /analytics/metrics/social-risk` — Расчет социального риска (погибшие на 100 тыс. населения).
31. `GET /analytics/metrics/transport-risk` — Расчет транспортного риска (погибшие на 10 тыс. ТС).
32. `GET /analytics/metrics/fatality-rate` — Коэффициент смертности на 100 пострадавших.
33. `GET /analytics/metrics/children-trauma-rate` — Удельный вес детского травматизма в общем числе ДТП.

#### Группа 7: Корреляционный анализ
34. `GET /analytics/correlation/weather-severity` — Зависимость тяжести последствий от погодных условий.
35. `GET /analytics/correlation/experience-accidents` — Анализ связи стажа вождения и вероятности попадания в ДТП.
36. `GET /analytics/correlation/lighting-time` — Зависимость количества ДТП от времени суток и освещения.
37. `GET /analytics/correlation/road-defects` — Доля ДТП из-за неудовлетворительного состояния дорог.

#### Группа 8: Сводные аналитические отчеты
38. `GET /analytics/reports/regional-dashboard` — Полный пакет данных для регионального дашборда (сводка индексов).
39. `GET /analytics/reports/participant-profile` — Аналитический портрет типичного виновника ДТП.
40. `GET /analytics/reports/hotspots-map` — Данные для тепловой карты (расчет плотности происшествий).
