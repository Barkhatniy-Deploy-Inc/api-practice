# Техническое задание: Зебра-стат API (Python + FastAPI)

## 1. Общее описание
Этот проект\u00A0— Enterprise-grade API (профессиональный программный интерфейс) для\u00A0работы со\u00A0статистикой ДТП. 
**Стек (набор технологий):** Python 3.11+, FastAPI (фреймворк для\u00A0API), Pydantic (валидация данных), Uvicorn (сервер).

---

## 2. Архитектурные требования
*   **Автодокументация:** доступна по\u00A0адресу `/docs` (интерфейс Swagger UI\u00A0для\u00A0живого тестирования).
*   **Безопасность:** проверка заголовка `X-API-KEY` (статический ключ доступа).
*   **Логирование:** Middleware (промежуточный слой кода), записывающий каждый запрос в\u00A0файл `logs/api_access.log`.
*   **Валидация (проверка):** строгая типизация данных через\u00A0Pydantic-схемы (модели данных).

---

## 5. Спецификация данных

### 5.1. Входные данные (Query Parameters\u00A0— параметры в\u00A0URL)
Основано на\u00A0реальных фильтрах системы stat.gibdd.ru.
Для\u00A0GET-запросов (получение данных) коллекции `/accidents` доступны параметры:
*   **limit** / **offset**: Pagination (постраничный вывод данных);
*   **period_start** / **period_end** (YYYY-MM): фильтрация по\u00A0периоду (месяцам);
*   **region_code** (string): код субъекта (например, «77» для\u00A0Москвы);
*   **district_code** (string): код федерального округа (ЦФО, ЮФО);
*   **indicator** (enum\u00A0— список значений): специфический показатель (например, `driver_fled`\u00A0— водитель скрылся);
*   **format** (enum: `json`, `csv`): Format (способ представления данных).

### 5.2. Структура JSON-ответа (Карточка ДТП)
Модель данных полностью соответствует официальной спецификации ГИБДД (opendataapi/v1/kartdtp).

```json
{
  "empt_number": "140000008",        // Уникальный номер ДТП
  "dtpv": "Наезд на пешехода",       // Вид происшествия
  "coord_w": "50.406494",            // Широта (гео-координата)
  "coord_l": "36.833982",            // Долгота (гео-координата)
  "location": {
    "district": "ШЕБЕКИНСКИЙ РАЙОН",
    "np": "Шебекино",
    "dor": "а/д Белгород-Шебекино"
  },
  "outcome": { "pog": 0, "ran": 1 }, // Погибшие и раненые
  "ts_info": [                       // Транспортные средства
    {
      "marka_ts": "RENAULT",
      "ts_uch": [                    // Участники ДТП
        {
          "kt_uch": "Водитель",
          "alco": "Нет опьянения",
          "v_st": 12                 // Стаж вождения
        }
      ]
    }
  ]
}
```

## 6. Реестр эндпоинтов (Всего: 40)

Эндпоинты (конечные точки API) разделены на\u00A0базовые данные и\u00A0сложные аналитические показатели (математика).

---

### 📦 КАТЕГОРИЯ А: Прямые данные (Direct Data)
*Данные, предоставляемые источником (ГИБДД/Росстат) в\u00A0сыром или\u00A0базовом виде.*

#### Группа 1: Справочники и\u00A0метаданные
1.  `GET /dictionaries/regions`\u00A0— список субъектов РФ.
2.  `GET /dictionaries/indicators`\u00A0— классификатор показателей аварийности.
3.  `GET /dictionaries/weather`\u00A0— виды погодных условий.
4.  `GET /dictionaries/road-conditions`\u00A0— состояние дорожного покрытия.
5.  `GET /dictionaries/accident-types`\u00A0— виды происшествий.
6.  `GET /metadata/source-info`\u00A0— сведения о\u00A0дате обновления и\u00A0объеме данных.

#### Группа 2: Детализированные списки
7.  `GET /accidents`\u00A0— полный список ДТП (с\u00A0фильтрами).
8.  `GET /accidents/{id}`\u00A0— полная карточка конкретного ДТП.
9.  `GET /accidents/by-region/{reg_code}`\u00A0— выборка ДТП по\u00A0конкретному региону.
10. `GET /accidents/by-period`\u00A0— выборка ДТП за\u00A0указанный промежуток времени.
11. `GET /accidents/by-indicator`\u00A0— ДТП, отфильтрованные по\u00A0одному показателю.
12. `GET /accidents/with-children`\u00A0— выборка всех ДТП с\u00A0участием несовершеннолетних.
13. `GET /accidents/drunk-drivers`\u00A0— выборка ДТП с\u00A0участием водителей в\u00A0состоянии опьянения.
14. `GET /accidents/fled-drivers`\u00A0— ДТП, с\u00A0места которых водитель скрылся.

#### Группа 3: Базовая статистика (Счетчики)
15. `GET /stats/basic/total-count`\u00A0— общее количество ДТП за\u00A0период.
16. `GET /stats/basic/total-pog`\u00A0— общее количество погибших.
17. `GET /stats/basic/total-ran`\u00A0— общее количество раненых.
18. `GET /stats/basic/by-district`\u00A0— общие цифры в\u00A0разрезе федеральных округов.
19. `GET /stats/basic/populated-areas`\u00A0— количество ДТП в\u00A0городах\u00A0vs сельская местность.
20. `GET /stats/basic/federal-roads`\u00A0— количество ДТП на\u00A0федеральных трассах.

---

### 🧮 КАТЕГОРИЯ Б: Расчетные показатели (Calculated Analytics)
*Показатели, получаемые путем математических вычислений, агрегации и\u00A0анализа на\u00A0стороне сервера.*

#### Группа 4: Тренды и\u00A0динамика
21. `GET /analytics/trends/monthly`\u00A0— процентное изменение аварийности месяц к\u00A0месяцу.
22. `GET /analytics/trends/yearly`\u00A0— динамика изменений показателей по\u00A0годам.
23. `GET /analytics/trends/prediction`\u00A0— прогноз аварийности на\u00A0следующий месяц (линейная экстраполяция).
24. `GET /analytics/seasonality`\u00A0— коэффициент сезонности (влияние времен года на\u00A0аварийность).

#### Группа 5: Сравнения и\u00A0рейтинги
25. `GET /analytics/compare/regions`\u00A0— сравнение показателей двух и\u00A0более регионов.
26. `GET /analytics/compare/periods`\u00A0— сравнение текущего периода с\u00A0АППГ (аналогичный период прошлого года).
27. `GET /analytics/rating/danger-zones`\u00A0— рейтинг самых опасных участков дорог.
28. `GET /analytics/rating/safe-regions`\u00A0— индекс безопасности регионов (взвешенный показатель).

#### Группа 6: Сложные коэффициенты и\u00A0математика
29. `GET /analytics/metrics/severity-index`\u00A0— расчет индекса тяжести последствий (погибшие\u00A0/ (погибшие\u00A0+\u00A0раненые)).
30. `GET /analytics/metrics/social-risk`\u00A0— расчет социального риска (погибшие на\u00A0100\u00A0тыс. населения).
31. `GET /analytics/metrics/transport-risk`\u00A0— расчет транспортного риска (погибшие на\u00A010\u00A0тыс. ТС).
32. `GET /analytics/metrics/fatality-rate`\u00A0— коэффициент смертности на\u00A0100\u00A0пострадавших.
33. `GET /analytics/metrics/children-trauma-rate`\u00A0— удельный вес детского травматизма в\u00A0общем числе ДТП.

#### Группа 7: Корреляционный анализ
34. `GET /analytics/correlation/weather-severity`\u00A0— зависимость тяжести последствий от\u00A0погодных условий.
35. `GET /analytics/correlation/experience-accidents`\u00A0— анализ связи стажа вождения и\u00A0вероятности попадания в\u00A0ДТП.
36. `GET /analytics/correlation/lighting-time`\u00A0— зависимость количества ДТП от\u00A0времени суток и\u00A0освещения.
37. `GET /analytics/correlation/road-defects`\u00A0— доля ДТП из-за\u00A0неудовлетворительного состояния дорог.

#### Группа 8: Сводные аналитические отчеты
38. `GET /analytics/reports/regional-dashboard`\u00A0— полный пакет данных для\u00A0регионального дашборда (сводка индексов).
39. `GET /analytics/reports/participant-profile`\u00A0— аналитический портрет типичного виновника ДТП.
40. `GET /analytics/reports/hotspots-map`\u00A0— данные для\u00A0тепловой карты (расчет плотности происшествий).
