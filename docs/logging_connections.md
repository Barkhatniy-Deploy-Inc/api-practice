# Спецификация логирования подключений (Connection Logging)

В\u00A0этом документе приведено детальное описание системы отслеживания входящих соединений и\u00A0сессий пользователей в\u00A0Зебра-стат API.

---

## 1. Цель системы
Логирование подключений предназначено для\u00A0мониторинга активности студентов, обеспечения безопасности сервиса и\u00A0анализа нагрузки на\u00A0API. Система позволяет точно идентифицировать, кто, когда и\u00A0с\u00A0какого устройства обращался к\u00A0серверу.

---

## 2. Объекты отслеживания
Система фиксирует следующие параметры для\u00A0каждого входящего HTTP-запроса:
*   **Временная метка (Timestamp):** точное время сервера с\u00A0миллисекундами.
*   **Идентификатор ключа (Prefix):** первые 8 символов API-ключа студента (сам ключ в\u00A0логи никогда не\u00A0записывается).
*   **IP-адрес клиента:** для\u00A0выявления подозрительной активности.
*   **Метод и\u00A0путь:** например, `GET /api/v1/analytics/summary`.
*   **User-Agent:** информация о\u00A0браузере или\u00A0библиотеке (Requests, cURL, Insomnia).

---

## 3. Формат записи (Structured Log)
Логи записываются в\u00A0формате JSON, что позволяет автоматически анализировать их внешними инструментами.

```json
{
  "timestamp": "2026-04-22 18:45:01,123",
  "level": "INFO",
  "type": "connection",
  "client": {
    "ip": "192.168.1.50",
    "user_agent": "python-requests/2.31.0"
  },
  "auth": {
    "key_prefix": "sk_dtp_a",
    "owner": "Иван Иванов"
  },
  "request": {
    "method": "GET",
    "path": "/api/v1/analytics/summary"
  }
}
```

---

## 4. Безопасность и\u00A0хранение
*   **Маскирование:** полные API-ключи никогда не\u00A0попадают в\u00A0файлы логов.
*   **Ротация:** файлы подключений ротируются каждые 10\u00A0МБ (хранится до\u00A010 архивных копий).
*   **Аудит:** попытки подключения с\u00A0невалидными или\u00A0заблокированными ключами помечаются уровнем `WARNING` и\u00A0содержат информацию об\u00A0источнике угрозы.

---

## 5. Использование в\u00A0Obsidian
Для\u00A0удобного анализа логов в\u00A0Obsidian рекомендуется использовать плагин **Dataview** для\u00A0построения графиков активности студентов на\u00A0основе JSON-экспорта.
