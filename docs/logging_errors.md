# Спецификация логирования ошибок (Error Logging)

В\u00A0данном документе описаны правила фиксации и\u00A0анализа нештатных ситуаций, возникающих при\u00A0работе Зебра-стат API.

---

## 1. Назначение
Логирование ошибок является критически важным инструментом для\u00A0поддержания стабильности образовательного сервиса. Оно позволяет быстро диагностировать проблемы в\u00A0базе данных, ошибки в\u00A0математических расчетах и\u00A0некорректные запросы клиентов.

---

## 2. Классификация фиксируемых ошибок
Система разделяет ошибки на\u00A0три уровня критичности:

### 🔴 Уровень ERROR (Критично)
*   **Сбои БД:** ошибки подключения к\u00A0SQLite, отсутствие таблиц или\u00A0колонок.
*   **Исключения в\u00A0коде (500):** необработанные Python-исключения.
*   **Ошибки аналитики:** деление на\u00A0ноль, ошибки в\u00A0NumPy или\u00A0матричных вычислениях.

### 🟡 Уровень WARNING (Предупреждение)
*   **Ошибки доступа (401, 403):** попытки использования неверных или\u00A0просроченных токенов.
*   **Нарушение лимитов (429):** слишком частые запросы от\u00A0одного студента.
*   **Проблемы синхронизации:** временная недоступность портала ГИБДД.

### 🔵 Уровень INFO (Информация об\u00A0ошибках валидации)
*   **Ошибки формата (422):** некорректный тип данных в\u00A0запросе (например, строка вместо числа).
*   **Логические ошибки:** запрос данных за\u00A0будущие периоды.

---

## 3. Состав сообщения об\u00A0ошибке
Каждая запись об\u00A0ошибке содержит исчерпывающий контекст:
*   `error_type`: класс исключения в\u00A0Python.
*   `request_id`: уникальный ID запроса для\u00A0сквозного поиска в\u00A0логах подключений.
*   `endpoint`: путь, на\u00A0котором произошел сбой.
*   `traceback`: сокращенный стек вызовов (только для\u00A0уровня ERROR).
*   `student_id`: идентификатор студента (если он был авторизован).

---

## 4. Пример структуры JSON
```json
{
  "timestamp": "2026-04-22 20:15:33,001",
  "level": "ERROR",
  "request_id": "req-9b2f",
  "message": "Database lookup failed: column 'is_intoxication' missing",
  "context": {
    "table": "participants",
    "operation": "select_detail"
  },
  "exception": "sqlalchemy.exc.OperationalError"
}
```

---

## 5. Ротация и\u00A0мониторинг
Ошибки записываются в\u00A0файл `logs/api_errors.log`. При\u00A0появлении более 5 критических ошибок за\u00A0минуту, система может быть настроена на\u00A0отправку уведомления администратору.

---

## 6. Рекомендации для\u00A0студентов
Если вы получили ответ со\u00A0статусом 500, пожалуйста, скопируйте `request_id` из\u00A0тела ответа и\u00A0передайте его преподавателю для\u00A0анализа логов ошибок.
