# 🚩 Жесткая критика архитектуры БД: Проблема «Водителей-младенцев»

Этот документ — официальный ответ бэкенд-разработчика на предложение оставить `DEFAULT 0` для поля возраста (`age`) в базе данных.

## 🛑 Почему это НЕ «идеальное состояние», а катастрофа

Ваша нейронка предлагает использовать костыль уровня «лишь бы не упало». Вот почему её логика — это профнепригодность в контексте API **аналитики**:

### 1. Смерть аналитики (NULL vs 0)
В анализе данных есть гигантская разница между **«значение неизвестно»** и **«значение равно нулю»**.
*   **0** — это возраст новорожденного ребенка.
*   **NULL** — это отсутствие информации.

Если мы оставим `DEFAULT 0`, то при первом же запросе аналитики (например, «зависимость ДТП от возраста») система выдаст бред: **новорожденные дети массово водят машины со стажем 17 лет и скрываются с места ДТП**. Средний возраст водителя в отчетах упадет до абсурдных значений. Мы будем врать пользователям API.

### 2. Подмена понятий и «магические числа»
Совет «Бэкендер привык писать `if age == 0: return null`» — это перекладывание мусора из одной корзины в другую. 
*   **Бэкенд не должен заниматься гаданием**, что вы имели в виду под числом 0. 
*   Если мы напишем такой `if`, мы **удалим реальные данные**. Что если в ДТП реально пострадал ребенок, которому 0 лет (младенец)? Ваш костыль превратит его в `null`, и мы потеряем данные о детском травматизме. Вы предлагаете уничтожать часть реальности ради того, чтобы ваш скрипт ленился вставлять `None`?

### 3. Нарушение стандартов SQL
Современные СУБД (даже SQLite) созданы для работы с `NULL`. Любой аналитик напишет `WHERE age IS NOT NULL` и получит чистую выборку. С вашими «нулями» аналитику придется писать костыли для фильтрации «ненастоящих нулей», постоянно рискуя ошибиться.

---

## 🛠 Что СРОЧНО нужно исправить (ТЗ для «базовика»):

1.  **Убрать `DEFAULT 0`** для колонки `age` (и других полей, где 0 может быть реальным значением).
2.  Сделать поля **`NULLABLE`**. Если данных нет — в базе должен быть `NULL`.
3.  В скрипте загрузки (`sync_gibdd.py`) вместо ленивого `val or 0` писать: 
    `val if val and str(val).isdigit() else None`.

## Итог
База данных — это фундамент. Если фундамент забит «навозом» в виде магических нулей, всё здание API рухнет при первой же серьезной агрегации данных. 

**Либо мы делаем качественный продукт с корректной обработкой NULL, либо не называем это «идеальным состоянием».**
